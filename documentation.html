Chapter 1

Introduction

1.1 Purpose

The purpose of Mediloop is to address the critical gap in personal medicine management by providing a unified, intelligent, and offline-capable platform. Millions of individuals struggle to track medication schedules, monitor expiry dates, and manage unused medicines responsibly. Mediloop resolves these challenges through a feature-rich web application that empowers patients, enables trusted NGOs and hospitals to receive medicine donations, and gives administrators full oversight of the ecosystem.

The system serves three primary objectives:

  • Medication Adherence: Ensure patients take the right medicine at the right time through smart reminders and dashboards.
  • Expiry Management: Reduce medicine wastage with color-coded visual alerts and timely notifications.
  • Community Donation: Bridge the gap between surplus medicine holders and certified NGOs or hospitals that distribute to those in need.

This document serves as a complete technical and functional reference for the Mediloop project, covering system design, development, testing, and deployment.

1.2 Scope

Mediloop is a Progressive Web Application (PWA) built with an offline-first architecture, ensuring usability even in low-connectivity environments. The scope of the system encompasses:

  • Patient Module: Medicine inventory management, OCR scanning, schedule reminders, donation requests, health records, AI assistant, and report generation.
  • NGO / Hospital Module: Viewing donation offers, confirming pickups, tracking received medicines, and maintaining distribution records.
  • Admin Module: User management, platform analytics, notification configuration, system settings, and data security oversight.
  • AI Integration: Context-aware health suggestions powered by the Gemini API.
  • Cloud Sync: Optional Firebase Firestore synchronisation for multi-device access.
Note: The platform is designed primarily for web browsers (Chrome, Firefox, Edge, Safari). A companion Android application is planned as a future enhancement.
Chapter 2

System Study and Analysis

2.1 Preliminary Investigation

The preliminary investigation was undertaken to evaluate the need for a centralised medicine management system. Key findings from the investigation are summarised below:

Finding Observation
Medicine wastage Approximately 30–40% of household medicines expire unused each year due to lack of tracking systems.
Missed doses Non-adherence to medication schedules accounts for significant health complications in chronic-disease patients.
Donation barriers Patients willing to donate surplus medicines have no reliable or certified channel to do so.
Elderly usability Existing apps are often not optimised for elderly users who need larger fonts and simpler navigation.

Based on this investigation, the project team concluded that a dedicated platform combining medicine management and donation coordination would provide measurable social and health benefits.

2.2 Existing System

Existing solutions in the market include general reminder apps and standalone pharmacy management tools. Their limitations are:

  • No integration between medicine tracking and donation features.
  • No OCR-based medicine data extraction.
  • Limited or no offline functionality.
  • No connection to verified NGOs or hospitals for ethical donation.
  • Absence of AI-powered health guidance personalised to the user's medicine list.
  • Poor accessibility for elderly or differently-abled users.

2.3 Proposed System

Mediloop proposes a holistic solution that combines all the above missing capabilities into one cohesive web platform. The system features an offline-first architecture using IndexedDB (via Dexie.js), optional cloud synchronisation through Firebase Firestore, and an intuitive UI built with React 18 and TailwindCSS.

2.3.1 Advantages of Proposed System

  • Offline-First: Full functionality without internet using IndexedDB.
  • OCR Scanning: Tesseract.js powered medicine strip scanning for quick data entry.
  • Color-Coded Expiry Alerts: Green / Yellow / Red / Gray visual system to prioritise action.
  • Donation Ecosystem: Verified NGO and hospital network for ethical medicine redistribution.
  • AI Health Assistant: Personalised guidance using the Gemini API.
  • Elderly-Friendly Mode: Enlarged UI, voice commands, and simplified navigation.
  • End-to-End Encryption: AES encryption ensures patient data privacy.
  • PWA Support: Installable on desktop and mobile for near-native experience.

2.4 Module Description

Patient Module

The Patient module is the core of Mediloop. It allows registered patients to:

  • Maintain a personalised medicine inventory with category tagging.
  • Scan medicine strips using the device camera (OCR).
  • Set daily/weekly dosage schedules with browser push notifications.
  • View color-coded expiry status and receive refill alerts.
  • Initiate donation requests to nearby NGOs or hospitals.
  • Generate PDF/Excel reports for doctor visits.
  • Access the AI Health Assistant for personalised suggestions.
  • Enable Elderly-Friendly Mode for an accessible interface.

NGO / Hospital Module

Certified NGOs and hospitals can:

  • Register with certification or license details for verification.
  • View and accept incoming medicine donation requests.
  • Schedule and confirm medicine pickups from patients.
  • Track received medicines and distribution analytics.
  • Communicate feedback to donating patients.

Admin Module

The Admin module provides platform-wide oversight:

  • Manage all registered users (patients, NGOs, hospitals).
  • View analytics dashboards for usage and donation trends.
  • Configure system-wide notifications and settings.
  • Verify NGO and hospital registrations.
  • Manage cloud sync and data security settings.

2.5 Feasibility Study

2.5.1 Technical Feasibility

The technology stack (React 18, TypeScript, Node.js, MongoDB, Firebase, Dexie.js, Tesseract.js) is mature, well-documented, and widely adopted. All components are open-source or have free tiers, making the system technically viable.

2.5.2 Operational Feasibility

The UI is designed for users with minimal technical knowledge, including an Elderly-Friendly Mode. Training requirements are minimal due to intuitive workflows and contextual tooltips.

2.5.3 Economic Feasibility

Development uses free and open-source tools. Firebase, MongoDB Atlas, and Vercel (hosting) each provide generous free tiers for the expected traffic of a community-scale deployment. Incremental paid tiers are available as the platform scales.

2.5.4 Behavioural Feasibility

The system introduces minimal change to users' existing behaviour — reminders, scanning, and dashboards align with common smartphone usage patterns. The donation feature provides positive social reinforcement, encouraging continued use.

2.5.5 Legal Feasibility

The platform complies with GDPR principles: data stays local by default, cloud sync is opt-in, no PII is shared with third parties, and all medical interactions are advisory (not prescriptive). Medicine donations are routed only to verified, government-certified NGOs and hospitals.

2.5.6 Schedule Feasibility

The project follows an agile development approach with three two-week sprints covering core features, donation workflow, and AI/reporting features respectively. The total timeline is approximately 8 weeks from inception to deployment.

Chapter 6

System Testing

6.1 White-Box Testing

Internal logic was tested with branch-coverage analysis on the expiry color-coding algorithm, donation status state machine, and OCR text parsing utility.

6.2 Black-Box Testing

All REST API endpoints were tested without knowledge of internal implementation, verifying HTTP status codes, response schemas, and error messages for: POST /auth/register, GET/POST/PUT/DELETE /medicines, POST /donations.

6.3 Condition Testing

Boolean conditions tested: user role checks (patient vs admin), medicine expiry comparisons (<30 / 30–90 / >90 days), and donation eligibility (quantity > 0 and not expired).

6.4 Loop Testing

Iterative routines verified: bulk NGO seeding loop, medicine list rendering with 0 / 1 / many items, and notification scheduling loop for each dose time.

One week after go-live: all three role flows verified in production; notifications confirmed on Chrome, Firefox, and Edge. One bug fixed — IndexedDB version conflict on Safari resolved by incrementing the DB version guard.

7.5 Security

  • Authentication: Firebase Auth with JWT tokens; sessions expire after inactivity.
  • Encryption: AES via crypto-js applied to sensitive local storage entries.
  • HTTPS: Enforced on all production routes (Vercel provides automatic TLS).
  • Role-Based Access: Route guards prevent cross-role page access.
  • CORS: Backend restricts origins to the deployed frontend domain only.

7.6 Maintenance

  • Automated dependency updates via Dependabot (GitHub).
  • Monthly review of Firebase and MongoDB Atlas quota usage.
  • Gemini API key rotation every 90 days.
  • Monitoring via Vercel Analytics and MongoDB Atlas Performance Advisor.

Conclusion

Mediloop successfully addresses a real-world healthcare challenge by combining medicine management and community donation into a single, accessible, and intelligent web platform. By leveraging React 18, TypeScript, Firebase, IndexedDB, and AI, the system delivers a reliable and privacy-respecting experience for patients, NGOs, hospitals, and administrators.

Key achievements:

  • Fully offline-functional medicine tracker with OCR scanning.
  • Verified donation ecosystem connecting patients to certified NGOs and hospitals.
  • AI-powered health assistant providing contextual medicine guidance.
  • Elderly-friendly mode ensuring inclusivity for all age groups.
  • End-to-end data security with AES encryption and Firebase Auth.
  • Progressive Web App architecture enabling near-native mobile experiences.

The system was developed following agile principles, thoroughly tested through white-box, black-box, integration, and user acceptance testing, and successfully deployed to a cloud environment. Mediloop demonstrates that technology can meaningfully reduce medicine wastage, improve medication adherence, and make healthcare resources more accessible to underserved communities.

import { useState } from 'react';
import { useNavigate, Link } from 'react-router-dom';
import { LogIn } from 'lucide-react';
import { UserRole } from '../../types';
import toast from 'react-hot-toast';
import { signInWithEmailAndPassword } from 'firebase/auth';
import { auth } from '../../config/firebase';

export default function Login() {
  const navigate = useNavigate();
  const [formData, setFormData] = useState({
    email: '',
    password: '',
    role: 'patient' as 'patient' | 'admin' | 'ngo' | 'hospital',
  });

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    try {
      await signInWithEmailAndPassword(auth, formData.email, formData.password);
      toast.success('Login successful!');
      navigate('/dashboard');
    } catch (error: any) {
      let errorMessage = 'Login failed. Please try again.';
      if (error.code === 'auth/invalid-credential')
        errorMessage = 'Invalid email or password. Please Sign Up first.';
      else if (error.code === 'auth/user-not-found')
        errorMessage = 'User not found. Please register.';
      toast.error(errorMessage);
    }
  };

  return (
    <div className="card">
      <h2 className="text-2xl font-bold text-center mb-6">Sign In</h2>
      <form onSubmit={handleSubmit} className="space-y-4">
        <div>
          <label className="block text-sm font-medium mb-2">Email</label>
          <input type="email" required className="input"
            value={formData.email}
            onChange={(e) => setFormData({ ...formData, email: e.target.value })}
            placeholder="you@example.com" />
        </div>
        <div>
          <label className="block text-sm font-medium mb-2">Password</label>
          <input type="password" required className="input"
            value={formData.password}
            onChange={(e) => setFormData({ ...formData, password: e.target.value })}
            placeholder="••••••••" />
        </div>
        <div>
          <label className="block text-sm font-medium mb-2">Role</label>
          <select className="input" value={formData.role}
            onChange={(e) => setFormData({ ...formData, role: e.target.value as UserRole })}>
            <option value="patient">Patient</option>
            <option value="ngo">NGO / Hospital</option>
          </select>
        </div>
        <button type="submit" className="btn btn-primary w-full">
          <LogIn size={18} />
          <span>Sign In</span>
        </button>
      </form>
      <p className="text-center mt-4 text-sm text-gray-600">
        Don't have an account?{' '}
        <Link to="/register" className="text-primary-600 font-medium">Sign Up</Link>
      </p>
    </div>
  );
}

REGISTER

File: src/pages/auth/Register.tsx

import { useState } from 'react';
import { useNavigate, Link } from 'react-router-dom';
import { UserPlus } from 'lucide-react';
import { db } from '../../db';
import toast from 'react-hot-toast';
import { generateId } from '../../utils/helpers';
import { createUserWithEmailAndPassword } from 'firebase/auth';
import { auth } from '../../config/firebase';

export default function Register() {
  const navigate = useNavigate();
  const [formData, setFormData] = useState({
    name: '', email: '', password: '', confirmPassword: '',
    role: 'patient' as 'patient' | 'admin' | 'ngo' | 'hospital',
  });

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    if (formData.password !== formData.confirmPassword) {
      toast.error('Passwords do not match');
      return;
    }
    try {
      const existing = await db.users.where('email').equals(formData.email).first();
      if (existing) { toast.error('Email already registered'); return; }

      // Create user in Firebase Authentication
      await createUserWithEmailAndPassword(auth, formData.email, formData.password);

      // Mirror user in local IndexedDB
      const newUser = {
        id: generateId('user-'),
        email: formData.email,
        name: formData.name,
        role: formData.role,
        createdAt: new Date(),
        updatedAt: new Date(),
        preferences: {
          theme: 'light' as const,
          elderlyMode: false,
          notificationsEnabled: true,
          voiceEnabled: false,
          language: 'en',
        },
      };
      await db.users.add(newUser);
      toast.success('Account created successfully!');
      navigate('/dashboard');
    } catch (error: any) {
      let msg = 'Registration failed. Please try again.';
      if (error.code === 'auth/email-already-in-use') msg = 'Email already in use';
      if (error.code === 'auth/weak-password')        msg = 'Password too short (min 6 chars)';
      toast.error(msg);
    }
  };

  return (
    <div className="card">
      <h2 className="text-2xl font-bold text-center mb-6">Create Account</h2>
      <form onSubmit={handleSubmit} className="space-y-4">
        <input type="text" required className="input" placeholder="Full Name"
          onChange={(e) => setFormData({ ...formData, name: e.target.value })} />
        <input type="email" required className="input" placeholder="Email"
          onChange={(e) => setFormData({ ...formData, email: e.target.value })} />
        <input type="password" required className="input" placeholder="Password"
          onChange={(e) => setFormData({ ...formData, password: e.target.value })} />
        <input type="password" required className="input" placeholder="Confirm Password"
          onChange={(e) => setFormData({ ...formData, confirmPassword: e.target.value })} />
        <select className="input"
          onChange={(e) => setFormData({ ...formData, role: e.target.value as any })}>
          <option value="patient">Patient</option>
          <option value="ngo">NGO / Hospital</option>
        </select>
        <button type="submit" className="btn btn-primary w-full">
          <UserPlus size={18} /><span>Create Account</span>
        </button>
      </form>
      <p className="text-center mt-4 text-sm">
        Already have an account? <Link to="/login">Sign In</Link>
      </p>
    </div>
  );
}

interface DashboardStats {
  totalUsers: number; totalMedicines: number; totalDonations: number;
  pendingDonations: number; completedDonations: number;
  activeUsers: number; systemHealth: 'good' | 'warning' | 'critical';
  recentActivity: number;
}

export default function AdminDashboard() {
  const [stats, setStats] = useState<DashboardStats>({
    totalUsers: 0, totalMedicines: 0, totalDonations: 0,
    pendingDonations: 0, completedDonations: 0,
    activeUsers: 0, systemHealth: 'good', recentActivity: 0,
  });
  const [recentUsers, setRecentUsers] = useState<User[]>([]);
  const [recentDonations, setRecentDonations] = useState<Donation[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => { loadDashboardData(); }, []);

  const loadDashboardData = async () => {
    setLoading(true);
    const users       = await db.users.toArray();
    const medicines   = await db.medicines.toArray();
    const donations   = await db.donations.toArray();
    const notifications = await db.notifications
                           .where('type').equals('system').toArray();

    const sevenDaysAgo = new Date();
    sevenDaysAgo.setDate(sevenDaysAgo.getDate() - 7);
    const activeUsers = users.filter(u => u.createdAt > sevenDaysAgo).length;

    const pendingDonations   = donations.filter(d => d.status === 'pending').length;
    const completedDonations = donations.filter(d => d.status === 'completed').length;

    let systemHealth: 'good' | 'warning' | 'critical' = 'good';
    if (pendingDonations > 50)  systemHealth = 'warning';
    if (pendingDonations > 100 || users.length === 0) systemHealth = 'critical';

    const oneDayAgo = new Date();
    oneDayAgo.setDate(oneDayAgo.getDate() - 1);
    const recentActivity = notifications.filter(n => n.createdAt > oneDayAgo).length;

    setStats({ totalUsers: users.length, totalMedicines: medicines.length,
      totalDonations: donations.length, pendingDonations, completedDonations,
      activeUsers, systemHealth, recentActivity });
    setRecentUsers(users
      .sort((a,b) => b.createdAt.getTime() - a.createdAt.getTime()).slice(0, 5));
    setRecentDonations(donations
      .sort((a,b) => b.createdAt.getTime() - a.createdAt.getTime()).slice(0, 5));
    setLoading(false);
  };

  // Returns JSX: stat cards (Users, Medicines, Donations, System Health),
  // recent users list, recent donations list, quick-action buttons,
  // and system notifications panel.
}

import { useEffect, useState } from 'react';
import { Pill, Calendar, Heart, AlertCircle, Plus, Camera } from 'lucide-react';
import { useAuthStore } from '../../store/useAuthStore';
import { db } from '../../db';
import { Medicine, MedicineSchedule } from '../../types';
import { getDaysUntilExpiry, formatDate } from '../../utils/helpers';
import { stockMonitoringService } from '../../services/stockMonitoringService';

export default function Dashboard() {
  const { user } = useAuthStore();
  const [medicines, setMedicines] = useState<Medicine[]>([]);
  const [schedules, setSchedules] = useState<MedicineSchedule[]>([]);
  const [stats, setStats] = useState({
    total: 0, expiringSoon: 0, todayDoses: 0,
    adherenceRate: 0, outOfStock: 0, lowStock: 0,
  });

  useEffect(() => {
    if (user) { loadData(); stockMonitoringService.checkAllNotifications(user.id); }
  }, [user]);

  const loadData = async () => {
    if (!user) return;
    const meds  = await db.medicines.where('userId').equals(user.id).toArray();
    const scheds = await db.schedules.where('userId').equals(user.id).toArray();
    setMedicines(meds); setSchedules(scheds);

    const expiringSoon = meds.filter(m => {
      const d = getDaysUntilExpiry(m.expiryDate); return d >= 0 && d <= 30;
    }).length;
    const todayDoses = scheds.reduce((c, s) => c + (s.times?.length || 0), 0);
    const stockSummary = await stockMonitoringService.getStockSummary(user.id);
    const adherenceRate = calculateAdherenceRate(scheds);
    setStats({ total: meds.length, expiringSoon, todayDoses,
               adherenceRate, ...stockSummary });
  };

  const calculateAdherenceRate = (schedules: MedicineSchedule[]): number => {
    if (!schedules.length) return 0;
    const now = new Date(), sevenDaysAgo = new Date(now);
    sevenDaysAgo.setDate(now.getDate() - 7);
    let totalDoses = 0, takenDoses = 0;
    schedules.forEach(schedule => {
      const start = new Date(schedule.startDate);
      const end   = schedule.endDate ? new Date(schedule.endDate) : now;
      for (let d = new Date(sevenDaysAgo); d <= now; d.setDate(d.getDate() + 1))
        if (d >= start && d <= end) totalDoses += schedule.times?.length || 0;
      schedule.taken?.forEach(r => {
        const td = new Date(r.date);
        if (td >= sevenDaysAgo && td <= now && r.taken) takenDoses++;
      });
    });
    return totalDoses === 0 ? 0 : Math.round((takenDoses / totalDoses) * 100);
  };

  // Returns JSX: stock alerts, stat cards (Total/Expiring/Doses/Adherence),
  // quick-action links (Add / Scan / Donate), upcoming doses list,
  // expiring medicines list, and AI-powered feature cards.
}

import { useEffect, useState } from 'react';

import { API_URL } from '../../config/api';

export default function NGODashboard() {
  const { user } = useAuthStore();
  const [stats, setStats] = useState({
    totalDonations: 0, pendingDonations: 0, completedDonations: 0,
    totalMedicines: 0, activeDonors: 0, thisMonthDonations: 0,
  });
  const [recentDonations, setRecentDonations] = useState<Donation[]>([]);
  const [loading, setLoading] = useState(true);
  const [ngoProfile, setNgoProfile] = useState<any>(null);

  useEffect(() => { if (user) loadDashboardData(); }, [user]);

  const loadDashboardData = async () => {
    setLoading(true);
    // Auto-register or fetch existing NGO profile
    const ngoRes = await fetch(`${API_URL}/api/ngos/${user?.id}`);
    if (ngoRes.ok) {
      setNgoProfile(await ngoRes.json());
    } else if (ngoRes.status === 404 && user) {
      await fetch(`${API_URL}/api/ngos`, {
        method: 'POST', headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ id: user.id, name: user.name,
          email: user.email, verified: true, location: { lat: 28.7041, lng: 77.1025 } }),
      });
    }

    // Fetch donations routed to this NGO
    const donationsRes = await fetch(`${API_URL}/api/donations/ngo/${user?.id}`);
    let donations: Donation[] = [];
    if (donationsRes.ok) {
      donations = (await donationsRes.json()).map((d: any) => ({
        ...d, createdAt: new Date(d.createdAt), updatedAt: new Date(d.updatedAt),
        medicines: d.medicines.map((m: any) => ({ ...m, expiryDate: new Date(m.expiryDate) }))
      }));
    }

    const pending   = donations.filter(d => d.status === 'pending').length;
    const completed = donations.filter(d => d.status === 'completed').length;
    const totalMeds = donations.reduce((s, d) => s + d.medicines.length, 0);

    const thirtyDaysAgo = new Date();
    thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30);
    const activeDonors = new Set(
      donations.filter(d => d.createdAt > thirtyDaysAgo).map(d => d.userId)
    ).size;

    const thisMonth = new Date(); thisMonth.setDate(1); thisMonth.setHours(0,0,0,0);
    const thisMonthDonations = donations.filter(d => d.createdAt >= thisMonth).length;

    setStats({ totalDonations: donations.length, pendingDonations: pending,
      completedDonations: completed, totalMedicines: totalMeds,
      activeDonors, thisMonthDonations });
    setRecentDonations(
      donations.sort((a,b) => b.createdAt.getTime()-a.createdAt.getTime()).slice(0,5));
    setLoading(false);
  };
  // Returns JSX: NGO info card, stat grid, urgent alerts, recent donations, quick actions.
}

import Dexie, { Table } from 'dexie';
import { User, Medicine, MedicineSchedule, Donation,
         NGO, Hospital, AppNotification,
         AnalyticsData, HealthRecord } from '../types';

export class MediloopDB extends Dexie {
  users!:         Table<User>;
  medicines!:     Table<Medicine>;
  schedules!:     Table<MedicineSchedule>;
  donations!:     Table<Donation>;
  ngos!:          Table<NGO>;
  hospitals!:     Table<Hospital>;
  notifications!: Table<AppNotification>;
  analytics!:     Table<AnalyticsData>;
  healthRecords!: Table<HealthRecord>;

  constructor() {
    super('MediloopDB');
    this.version(2).stores({
      users:         'id, email, role, createdAt',
      medicines:     'id, userId, name, category, expiryDate, createdAt',
      schedules:     'id, medicineId, userId, startDate, endDate',
      donations:     'id, userId, ngoId, hospitalId, status, createdAt',
      ngos:          'id, name, verified',
      hospitals:     'id, name, verified',
      notifications: 'id, userId, type, read, createdAt',
      analytics:     'id, userId, period, date',
      healthRecords: 'id, userId, type, date, createdAt',
    });
  }
}
